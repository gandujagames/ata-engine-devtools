# Contributing

Thanks for your interest in contributing.

This project accepts small, practical improvements related to game engine development workflows, build validation, asset validation, and log scanning.

## Contribution types

Useful contributions include:

- Bug fixes
- Documentation improvements
- New validation scripts
- Better error messages
- Cross-platform improvements
- GitHub Actions improvements

## Pull requests

Before opening a pull request:

1. Keep the change focused.
2. Explain what problem it solves.
3. Include a short usage example if applicable.
4. Avoid adding large dependencies unless necessary.

## Code style

Prefer simple, readable scripts that are easy for small teams to understand and modify.
